Green Week — One-Page Sprint Kit (print & grind)

Objective (7 days). Turn 100 failing PRs green using tiny patches (≤40 LOC, ≤5 files, ≤3 attempts, never touch main). Publish proof daily. Show HN at week’s end → ≥25 demos, ≥10 installs, pipeline to $1k MRR.

Hard Rules (don’t break these)

  • Patch budget: ≤40 LOC, ≤5 files, ≤3 attempts/PR.
  • Fork → nova/fix/<slug> branch; no pushes to main.
  • Respect CONTRIBUTING.md. Risky fixes → comment-only suggestion.
  • Every PR body includes rationale, diff stats, before/after, rollback plan.

Success Metrics & Gates

  • Daily throughput: 20 attempts / 8–12 opened PRs.
  • Quality: median LOC ≤25; p50 time-to-green (TTG) ≤60m.
  • Engagement: maintainer comments/acks ≥40% of opened PRs.
  • Demand: ≥4 inbound demos/day (2-day rolling).
  • Scale/stop: <2 demos/day ×2 → retarget; >6/day → pause outbound.
  • DoD (week): ≥60 accepted PRs · ≥20 maintainer comments · ≥25 demos · ≥10 installs · Show HN live w/ gallery + CTA.

language:Python OR language:TypeScript OR language:Go -label:WIP

Keep PRs with failing checks; prefer keywords: flake|flaky|timeout|seed|random|order|ImportError|ModuleNotFound|TypeError|AssertionError|lint|black|ruff|eslint|tsc|mypy|pytest. Stars 100–5000, active <90d, GitHub Actions first.

Ops & Conversion

  • CTA everywhere: “Book 15-min” or “Install GH App / start trial”.
  • Canned replies: thanks, security/compliance, access/permissions, guardrails. SLA <4h.
  • Daily metrics (6pm): attempts, accepted, median LOC, median TTG, maintainer comments, demos, installs.

Risks → Mitigations

  • Noisy fixes → enforce budgets; prefer comment-only; human review on borderline patches.
  • Rate limits → rotate tokens; cache logs; exponential backoff.
  • Community fatigue → rotate repos/stacks; avoid repeats; be respectful.
  • Launch stall → bank 2–3 strong rescues/day pre-launch; keep a queue.
